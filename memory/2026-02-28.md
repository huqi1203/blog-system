## 监控面板功能完善

### 问题诊断
大哥反馈局域网可以访问监控面板，但存在三个问题：
1. 系统资源内存使用没有数据显示
2. 并发控制不能手动调整
3. 并发槽位状态没有实时显示

### 解决方案

#### 1. 内存监控修复
**问题根因**：中文 Linux 系统上 `free` 命令输出的是 "内存：" 而不是 "Mem:"，原代码使用正则匹配 `awk '/^Mem:/{...}'` 无法匹配。

**修复方法**：改用行号匹配 `awk 'NR==2{print $2}'`，兼容所有语言环境。

**修复位置**：
- /tmp/openclaw-monitor/server.py
- /root/.openclaw/workspace/monitor-panel/server.py

**验证结果**：
```
总内存 (GB): 5
已用内存 (GB): 1
内存使用率: 34%
```

#### 2. 并发控制功能完善
**新增 API 接口**：
- POST /api/concurrent/increase - 增加并发槽位（可指定 delta）
- POST /api/concurrent/decrease - 减少并发槽位（可指定 delta）
- POST /api/concurrent/set - 设置并发槽位数（直接指定 value）
- GET /api/concurrent/status - 获取并发状态摘要
- GET /api/concurrent/slots - 获取详细的槽位使用情况

**功能特性**：
- 支持手动调整并发数（1-50 范围内）
- 自动备份配置文件到 /root/.openclaw/backups/
- 同时更新 maxConcurrent 和 subagents.maxConcurrent
- 实时返回操作结果

**验证结果**：
- 增加槽位：从 10 增加到 12 ✅
- 减少槽位：从 12 减少到 11 ✅
- 设置槽位：设置为 10 ✅
